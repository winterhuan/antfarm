## Key Findings

1. **step-ops.ts has clear domain boundaries**: Functions cluster into parsing (lines 24-270), step lifecycle (lines 467-959), and failure handling (lines 961-1103).
2. **Backend interface is too thin**: Only 4 methods, but backends have significantly different internal patterns.
3. **Error handling is inconsistent**: Mixes `throw new Error()`, `process.exit(1)`, silent `catch {}` blocks.
4. **Step entity has dual-mode lifecycle**: Single steps (linear) vs loop steps (per-story sub-lifecycles with verify-each).
